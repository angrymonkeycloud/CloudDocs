
# Angry Monkey Cloud Documentation Hub

This repository is the centralized documentation source for Angry Monkey teams and creators. It is structured for long-term growth, stable links, and clear discoverability.

## Documentation Principles

- Stable paths: avoid moving or renaming existing pages.
- One topic per page: easier updates, ownership, and search indexing.
- Clear hierarchy: each domain has an index page and focused subpages.
- SEO-first writing: clear titles, summaries, and keyword-aware headings.

## SEO Implementation Notes

- Every page starts with a descriptive title and summary paragraph.
- Domain index pages include internal links to related topics.
- Headings use high-intent phrases users are likely to search.
- Pages avoid vague names to improve crawl and ranking quality.

## Contribution Guidance

- Add new content under the existing domain folders.
- Prefer creating new pages instead of overloading existing ones.
- Keep paths permanent once published.
- Cross-link related pages to improve navigation and SEO.
