# File and Folder Standards

This page defines storage structure and handling rules for operational consistency.

## Documents Structure

### Client Name Folder

- Social Media
- Branding
- Development
- Received from Client

### Received from Client

- Use date-based folders in the format YYYY-MM-DD.

## Client Drive Handling

- Use the drive only as transfer space.
- Move received files into the internal document structure.
- Keep transfer folders clean and organized.

## Project Delivery Folder

- Keep final approved outputs in project delivery locations.
- Maintain read/write permissions according to role.

## Naming and Versioning Rules

- Do not delete files/folders created by other team members unless approved.
- Avoid naming suffixes such as final or amendments.
- Use date or explicit version number instead.
